# Articles App — Frontend

Фронтенд‑часть приложения для работы со статьями, версиями, вложениями и аутентификацией.  
Использует React, React Router и Axios.

## Технологии
- React (CRA)
- React Router v6
- Axios
- React Quill (редактор текста)
- WebSocket клиент
- JWT аутентификация

## Установка и запуск
Перейдите в папку `frontend` и установите зависимости:
```bash
cd frontend
npm install
npm start
```

## Аутентификация (JWT)
После успешного логина фронтенд сохраняет:
```bash
token
role
```
в localStorage.
Все защищённые запросы отправляются с заголовком: Authorization: Bearer <token>

## Роли пользователей (RBAC)
- user — обычный пользователь
- admin — администратор
Роль используется для:
- скрытия/показа элементов интерфейса
- ограничения доступа к страницам
- проверки прав на редактирование статей

## Защищённые маршруты
Используется компонент: <RequireAuth>
Он перенаправляет неавторизованных пользователей на /login.
Защищённые страницы:
- / — список статей
- /article/:id — просмотр статьи
- /create — создание статьи
- /edit/:id — редактирование статьи
- /admin/users — управление пользователями (только admin)

## Скрытие элементов интерфейса
- неавторизованных пользователей
- обычных пользователей
- страниц /login и /register

## Работа со статьями
- создание статьи
- редактирование статьи
- загрузку вложений
- просмотр версий
- переход к старым версиям
- запрет редактирования чужих статей
Редактирование отправляется на:
```bash
PUT /articles/:id/edit
```
Создание версии:
```bash
PUT /articles/:id
```

## Управление пользователями (admin)
Страница /admin/users доступна только администратору.
Функционал:
- просмотр всех пользователей
- изменение роли (user/admin)
- обновление списка без перезагрузки
